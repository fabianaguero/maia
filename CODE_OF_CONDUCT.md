# Code of Conduct

Maia should be a respectful place to collaborate.

## Expected Behavior

- Be direct, technical, and respectful.
- Assume good faith, but be precise when something is wrong.
- Focus criticism on code, architecture, product tradeoffs, and behavior, not on people.
- Help keep discussion useful for contributors with different experience levels.

## Unacceptable Behavior

- Harassment, threats, or intimidation
- Personal attacks or demeaning language
- Discrimination based on identity or background
- Repeated bad-faith disruption of issues, PRs, or discussions
- Publishing private information without permission

## Maintainer Standards

Maintainers may edit, hide, lock, or remove content that clearly violates these expectations.
They may also pause or close threads that are no longer productive.

## Reporting

If you experience or observe a serious conduct problem, contact the repository owner or maintainers privately through GitHub.
If a dedicated reporting contact is added later, this document should be updated to use it.

## Scope

This code of conduct applies to repository issues, pull requests, discussions, and other project-managed collaboration spaces.
